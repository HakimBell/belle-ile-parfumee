Choix de l'architecture monolithique

Le backend repose sur une architecture monolithique, adaptée au contexte du projet :

  • Projet de taille moyenne : plus simple à concevoir et maintenir
  • Développement en autonomie : pas de coordination entre équipes
  • Déploiement simplifié : un seul service à gérer
  • Bonnes performances : pas de latence réseau entre services

Docker et la conteneurisation

Docker permet de conteneuriser l'application, garantissant un fonctionnement identique en local et en production.

Dockerfile Backend (Multi-stage build)

FROM eclipse-temurin:21-jdk AS build WORKDIR /app COPY . . RUN ./mvnw clean package -DskipTests FROM eclipse-temurin:21-jre WORKDIR /app COPY --from=build /app/target/*.jar app.jar EXPOSE 8081 ENTRYPOINT ["java", "-jar", "app.jar"]

Docker Compose - Orchestration

services: postgres: # Base de données backend: # API Spring Boot frontend: # Application React
Commande : docker compose up --build lance les 3 conteneurs

Problèmes rencontrés et solutions

Problème 1 : Configuration CORS

Contexte : Le frontend ne pouvait pas communiquer avec le backend.

Solution : Ajout de l'URL Render dans les origines autorisées :

configuration.setAllowedOrigins(List.of( "http://localhost:5173", "http://localhost", "https://belle-ile-front.onrender.com" ));

Problème 2 : Routing SPA en production

Contexte : Les routes React affichaient "Not Found" après refresh.

Solution : Ajout d'un fichier _redirects :

/* /index.html 200
